# Glossary

This glossary explains common ClaudIA, AI, Microsoft 365, Azure, and security terms in beginner-friendly language.

## AI Concepts

| Term | Simple explanation |
| --- | --- |
| AI | Software that can generate, summarize, classify, reason, or assist with tasks that normally require human-like understanding. |
| Agent | A system or script that can perform actions for a user or workflow. In ClaudIA, agents simulate fictional employees. |
| Prompt | The instruction or question given to an AI system. |
| Response | The output generated by an AI system after receiving a prompt. |
| Context | The information available to the AI when generating a response. This may include the prompt, conversation history, selected files, retrieved data, or system instructions. |
| Session | A period of interaction with a system. Some systems keep temporary state during a session. |
| Memory | Information remembered across interactions. Not all AI systems have long-term memory. Workplace AI must be governed carefully when memory or history is involved. |
| Grounding | Connecting an AI response to specific data sources, documents, or context instead of relying only on the model's general knowledge. |
| Copilot | A Microsoft AI assistant experience integrated into Microsoft 365 and other Microsoft products. |
| Shadow AI | Use of AI tools that are not approved, governed, or monitored by the organization. |
| AI governance | Policies, controls, and practices that help an organization use AI safely and responsibly. |

## Microsoft 365 Concepts

| Term | Simple explanation |
| --- | --- |
| Microsoft 365 | Microsoft's cloud productivity platform, including Outlook, Teams, SharePoint, OneDrive, and other services. |
| Tenant | A company's dedicated Microsoft cloud environment. It contains users, groups, apps, policies, licenses, and data. |
| User | An account representing a person or persona in the tenant. |
| Persona | A fictional user used in ClaudIA to simulate workplace activity. |
| UPN | User Principal Name. Usually the sign-in name, such as `priya.sharma@contoso.example`. |
| Group | A collection of users used for access, policies, collaboration, or administration. |
| License | A subscription assignment that enables Microsoft 365 features for a user. |
| Outlook / Exchange Online | Microsoft email and calendar services. |
| Teams | Microsoft collaboration platform for chat, meetings, channels, and teamwork. |
| SharePoint Online | Microsoft platform for team sites, document libraries, and collaboration. |
| OneDrive for Business | Personal cloud file storage for work users. |
| Microsoft Lists | A Microsoft 365 app for structured lists and business tracking. |
| Microsoft Fabric | Microsoft analytics platform for data engineering, data science, and business intelligence scenarios. |

## Azure Concepts

| Term | Simple explanation |
| --- | --- |
| Azure | Microsoft's cloud computing platform. ClaudIA uses Azure to run automation, AI, APIs, telemetry, and storage. |
| Subscription | A billing and management boundary in Azure. Resources are created inside subscriptions. |
| Resource group | A container for Azure resources that belong together. |
| Azure Automation | Azure service that runs scheduled scripts and runbooks. |
| Runbook | A script or automation workflow executed by Azure Automation. |
| Managed identity | An Azure identity assigned to a service so it can access other resources without storing a password in code. |
| RBAC | Role-Based Access Control. A way to assign permissions to users, groups, apps, or managed identities. |
| Azure Key Vault | A secure Azure service for storing secrets, keys, and certificates. |
| Secret | A sensitive value such as a password, token, or client secret. |
| Azure Data Explorer | A fast analytics service used to store and query telemetry with KQL. Also called ADX. |
| KQL | Kusto Query Language. A query language used by Azure Data Explorer and several Microsoft security tools. |
| Azure Function | A small cloud-hosted API or function that runs code when called. |
| Storage Static Website | A low-cost way to host static web files such as HTML, CSS, JavaScript, and images. |
| Azure Front Door | Azure edge service used for public endpoints, custom domains, routing, and caching. |
| Azure OpenAI | Azure service that provides access to OpenAI models for approved workloads. |
| Azure AI Foundry | Azure platform for building, deploying, and managing AI applications and model scenarios. |

## Security And Governance Concepts

| Term | Simple explanation |
| --- | --- |
| Microsoft Purview | Microsoft platform for data security, compliance, information protection, DLP, labels, records, and governance. |
| Microsoft Defender | Microsoft security platform for detecting, investigating, and responding to threats. |
| MDCA | Microsoft Defender for Cloud Apps. Helps discover, monitor, and govern cloud app usage. |
| Microsoft Sentinel | Microsoft's cloud-native SIEM and SOAR platform for security monitoring and response. |
| DLP | Data Loss Prevention. A control that helps detect or prevent sensitive data from being shared or used in unsafe ways. |
| Sensitivity label | A label applied to content to identify and protect information, such as Public, Confidential, or Highly Confidential. |
| Insider Risk | Risk caused by users inside the organization, whether intentional, accidental, or negligent. |
| DSPM | Data Security Posture Management. Helps understand where sensitive data exists and how exposed it may be. |
| DSPM for AI | Data security posture concepts applied to AI use, prompts, responses, and AI-accessible data. |
| Audit log | A record of actions that happened in a system. |
| Telemetry | Event data created by systems and users. It helps with monitoring, investigation, and analysis. |
| Detection | A rule or process that identifies suspicious, risky, or important activity. |
| Investigation | The process of understanding what happened, who was involved, what data was affected, and what should be done. |
| Exfiltration | Unauthorized or risky movement of data out of an approved location. |
| Oversharing | Giving more people or systems access to information than intended or necessary. |
| Conditional Access | Microsoft Entra policy system that controls access based on conditions such as user, device, location, risk, or MFA. |
| MFA | Multi-Factor Authentication. A sign-in method that requires more than just a password. |
| Security Defaults | Microsoft Entra baseline security settings that enforce basic protections such as MFA. Good for real tenants, but often incompatible with unattended lab automation. |
| ROPC | Resource Owner Password Credentials. A legacy OAuth flow where a username and password are exchanged for a token. ClaudIA uses it only for lab simulation. |

## ClaudIA-Specific Concepts

| Term | Simple explanation |
| --- | --- |
| ClaudIA | Cloud Activity, Usage & Data Intelligence Architecture. The open source platform and project. |
| MH Demos | The fictional company and tenant used to demonstrate one ClaudIA implementation. |
| Activity Story Map | ClaudIA's visual portal for exploring personas, services, architecture, and activity. |
| BrowserAgents | Playwright-based browser automation that simulates user activity through web interfaces. |
| Synthetic content | Fictional documents, messages, prompts, and records generated for safe lab scenarios. |
| Controlled risky activity | Fictional activity that may look suspicious or incorrect, created intentionally to teach security concepts. |
| Public-safety check | A repository validation step that helps detect secrets, generated files, or unsafe values before publication. |

## Beginner Tip

When you see a new term, ask three questions:

1. Is this about identity, data, AI, security, or infrastructure?
2. Is it a Microsoft 365 service, an Azure service, or a security concept?
3. What role does it play in the ClaudIA story?
