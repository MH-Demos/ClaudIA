# Personas

ClaudIA uses fictional personas to make Microsoft 365 security and governance easier to understand.

A normal lab tenant is often empty. It may contain services, policies, and configuration, but it does not feel like a company. ClaudIA solves that by creating synthetic users who work, collaborate, send messages, create files, interact with AI, and generate security-relevant telemetry.

These personas are not decoration. They are the human layer of the demo.

## Why Personas Matter

Security tools become easier to explain when the audience can answer five questions:

1. Who acted?
2. What did they touch?
3. Which service was involved?
4. What risk or signal was created?
5. Which security or governance control helps explain the result?

ClaudIA personas help answer those questions across Microsoft 365, Purview, Defender, ADX, browser automation, and the Activity Story Map.

## Devon Reyes And Risk Simulation

Devon Reyes has a special role in the ClaudIA storyline.

Devon is not an external attacker. Devon is a fictional internal user used to generate activity that may be considered risky, suspicious, incorrect, or policy-violating from a security perspective.

Use Devon to explain scenarios such as:

- Uploading or sharing sensitive content in the wrong place.
- Using external AI tools with sensitive business information.
- Creating unusual data movement patterns.
- Triggering DLP, Insider Risk, or investigation workflows.
- Showing why user behavior, permissions, and AI governance must be understood together.

This makes Devon useful for demos about Shadow AI, oversharing, insider risk, data loss, and security operations.

## How Personas Connect To ClaudIA Components

| Component | Persona relationship |
| --- | --- |
| `config/agents.json` | Defines persona identity, workload focus, working hours, topics, licensing, and secret names. |
| `Storyline/profiles.md` | Maintains the detailed public profile for each persona. |
| `Images/Characters` | Stores persona images used by the Activity Story Map and Microsoft 365 profile photo upload scripts. |
| Activity Story Map | Shows users, relationships, activity, services, and telemetry visually. |
| Azure Data Explorer | Stores normalized activity generated by personas. |
| Microsoft Purview and Defender | Surface the security and governance signals created by persona activity. |

## Adding A New Persona

When adding a new persona:

1. Add the persona to `config/agents.json`.
2. Add or update the profile in `Storyline/profiles.md`.
3. Add the persona image under `Images/Characters` using a predictable display-name-based filename.
4. Update this guide if the persona changes the learning path or story model.
5. Update any scenario, team, manager, or storyline reference that depends on the persona.
6. Republish Activity Story Map assets if the portal is enabled.

## Important Safety Rule

All personas must remain fictional. Do not use real employee names, real UPNs, real photos, real customer data, real production documents, or real confidential information in a public ClaudIA deployment.
